# Plan Export for External AI Agents
この会話セッションでこれまで議論した調査・設計・タスク内容を、他のAIコーディングエージェント（特にCodex CLI）が実行できる形式のプロンプトに変換してください。

## 変換ルール
この会話の内容を以下の構造で整理し、Markdown形式で出力してください：

### 1. 概要（Background）
- 何を実現したいのか
- なぜこの実装が必要なのか
- 現在の状態と目標の状態

### 2. 調査結果（Investigation）
- 関連するファイル・コード箇所
- 既存の実装パターン
- 技術的な制約や考慮事項

### 3. 設計方針（Design）
- 採用するアプローチとその理由
- アーキテクチャ上の決定事項
- 代替案を選ばなかった理由（あれば）

### 4. 実装タスク（Tasks）
- 番号付きの実装ステップ
- 各タスクの依存関係
- 完了条件

### 5. 参照ファイル（References）
- 変更対象のファイルパス
- 参考にすべき既存コード
- テスト要件

## 出力要件
1. **自己完結**: このプロンプトだけで実装可能な情報を含める
2. **具体的**: ファイルパス、関数名など具体的に記載
3. **順序明確**: タスクの実行順序が明確
4. **コピー可能**: そのままCodex CLIに渡せる形式

## 出力フォーマット
以下の形式で出力してください。コードブロック内がコピー対象です：
```markdown
# [タスク名]

## Background
[概要を記載]

## Investigation Results
[調査結果を記載]

## Design Decisions
[設計方針を記載]

## Implementation Tasks
1. [タスク1]
2. [タスク2]
...

## File References
- `path/to/file.ts`: [説明]
- ...

## Constraints
- [制約事項があれば記載]
```
